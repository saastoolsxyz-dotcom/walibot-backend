Connect Your Frontend

Use these endpoints to build your own custom React, Vue, or Next.js app on top of WaliBot.

Base Production URL

GET /status

Returns real-time operational state including message counts and unique leads.

{
  "stats": { "totalMessages": 142, "pendingOrders": 3, "uniqueUsers": 21 },
  "business": { "name": "Wali Store", ... }
}
POST /settings

Update bot configuration such as response delay and API provider settings.

{
  "responseDelay": { "min": 2, "max": 5 },
  "provider": "groq"
}
POST /webhook/order

External CRM integration. Send order updates from your store to trigger WhatsApp notifications.

{
  "customer_phone": "923xxx",
  "status": "confirmed", // or processing, completed, cancelled
  "order_number": "12345",
  "customer_name": "Abdullah"
}

Frontend AI Builder Prompt

Copy this prompt and paste it into any AI coding tool (like v0.dev or Claude) to generate a full React frontend that connects to this backend instantly.

ACT AS A SENIOR REACT DEVELOPER. 
Build a clean, high-converting React Dashboard for a WhatsApp AI SaaS called "WaliBot".

INTEGRATION SPECS:
- Base API URL: ${window.location.origin}
- Use 'fetch' for API calls.
- Main endpoints:
  1. GET /status (System health & stats)
  2. GET /dashboard-data (Orders, Lead lists, AI Questions)
  3. POST /confirm-order { id, confirmedDetails } (Confirm a sale)
  4. POST /start-bot { businessName, websiteUrl, ... } (Launch session)

UI REQUIREMENTS:
- Primary Color: Emerald-500 (#10b981)
- Dark Theme Layout
- Tabs for: "Sales Pipeline", "Customer Chats", "Bot Intelligence"
- A prominent 'Link WhatsApp' QR section that polls /qr
- A 'Real-time Feed' that lists messages from /messages

FUNCTIONAL GOAL:
The app must allow business owners to link their WhatsApp, train their bot by answering questions, and confirm orders with a single click.
